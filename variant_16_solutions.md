## Задание 1: Выражение на C

**Формула:**
$$ \sqrt{\frac{4|x| - a}{x^2 - 2}} + \cot(x) - \frac{2}{3} $$

**Синтаксис C:**
```c
sqrt((4 * fabs(x) - a) / (x * x - 2)) + 1.0 / tan(x) - 2.0 / 3.0
```

**Примечания:**
- `sqrt(...)` оборачивает первую дробь.
- `fabs(x)` используется для модуля (требует `<math.h>`).
- `tan(x)` используется, так как `cot(x) = 1/tan(x)`.
- `2.0 / 3.0` гарантирует деление с плавающей точкой.
- Скобки минимизированы, используются только там, где это необходимо для группировки числителей/знаменателей.

## Задание 2: Условный оператор (Блок-схема в код)

**Отображение логики:**

```c
if (z) {
    if (x) {
        c = 1;
    }
} else {
    v = 5;
}
```

**Пояснение:**
- **Внешнее условие `z`:**
  - **TRUE (Да):** Входит в блок проверки `x`.
  - **FALSE (Нет):** Переходит в `else` и выполняет `v = 5;`.
- **Внутреннее условие `x` (Вложено в `z`=TRUE):**
  - **TRUE (Да):** Выполняет `c = 1;`.
  - **FALSE (Нет):** Пустая ветка (Ничего не делать).

## Задание 3: Протокол выполнения (Трассировка)

**Логика кода:**
- `a = {1, -7, -5, 2, -1}` (Размер N=5)
- Цикл выполняется пока `i < 5` И `k < 2`.
- `k` увеличивается, если `a[i] < 0`.

**Таблица трассировки:**

| Шаг | i | k (нач) | a[i] | Условие `a[i] < 0` | k (кон) | Проверка цикла `i<N && k<2` |
| :--- | :-: | :-: | :-: | :--- | :-: | :--- |
| **Инит** | 0 | 0 | - | - | 0 | **TRUE** (0<5 && 0<2) |
| **1** | 0 | 0 | `1` | False | 0 | **TRUE** (1<5 && 0<2) |
| **2** | 1 | 0 | `-7` | **True** | 1 | **TRUE** (2<5 && 1<2) |
| **3** | 2 | 1 | `-5` | **True** | 2 | **FALSE** (3<5, но 2<2 ложно) |

**Результат:**
Цикл завершается, так как `k` достигло 2.
- **Условие вывода:** `if (k==2)` — **True**.
- **Печать:** `i-1`, `a[i-1]` -> `3-1`, `a[3-1]` -> `2`, `a[2]`.
- Значение `a[2]` равно `-5`.

**Итоговый вывод:**
```text
2, -5
```

## Задание 4: Анализ сортировки

**Анализ:**
- **Метод:** **Сортировка выбором (Selection Sort)**. (Выбирает экстремальный элемент и помещает его на текущую позицию `i`).
- **Логика:** `if (a[j] > r)` проверяет значения, которые *больше* текущего выбранного элемента.
- **Направление:** **По убыванию** (от большего к меньшему). Самый большой оставшийся элемент находится и перемещается в начало (индекс `0`, затем `1` и т.д.).

**Выполнение пошагово:**
1.  **Проход i=0**: Ищет максимум в `a[0]...a[4]`. Максимум `'e'` (индекс 1). Менят `a[0]` ('a') и `a[1]` ('e'). Массив: `e, a, c, b, d`.
2.  **Проход i=1**: Ищет максимум в `a[1]...a[4]`. Максимум `'d'` (индекс 4). Менят `a[1]` ('a') и `a[4]` ('d'). Массив: `e, d, c, b, a`.
3.  **Проход i=2**: Ищет максимум в `a[2]...a[4]`. Максимум `'c'` (индекс 2). Без изменений. Массив: `e, d, c, b, a`.
4.  **Проход i=3**: Ищет максимум в `a[3]...a[4]`. Максимум `'b'` (индекс 3). Без изменений. Массив: `e, d, c, b, a`.

**Итоговый вывод:**
```text
  e,  d,  c,  b,  a,
```

## Задание 5: Визуализация сектора матрицы

**Условия:**
1.  `i + j < n - 1`: **Строго выше побочной диагонали**.
2.  `i - j >= 0` $\rightarrow$ `i >= j`: **Ниже или на главной диагонали**.

**Пересечение (пример 5x5):**
- **Строка 0:** `(0,0)` (Оба условия верны).
- **Строка 1:** `(1,0)`, `(1,1)`.
- **Строка 2:** `(2,0)`, `(2,1)`.
- **Строка 3:** `(3,0)`.
- **Строка 4:** Нет.

**Графическая визуализация:**
( `X` = Выбрано, `.` = Не выбрано )

```text
X . . . .  <- (0,0)
X X . . .  <- (1,0), (1,1)
X X . . .  <- (2,0), (2,1)
X . . . .  <- (3,0)
. . . . .
```

**Геометрическое описание:**
**Левый треугольник** (Западный сектор) матрицы, включая сегменты главной диагонали в этих границах, но строго исключая побочную диагональ.
